# EchoVessel Documentation

> **Maintainer rule**: in any `.md` file, link to architecture HTML pages using the published URL `https://alanyian.com/projects/echovessel/docs/<file>.html`, not the repo path. GitHub renders `.md` but serves raw source for `.html`, so a repo-relative link drops readers into source view. HTML-to-HTML internal links (e.g. `architecture-flow.html` → `./architecture.html`) stay relative — they resolve correctly both on the published site and in a local clone.
